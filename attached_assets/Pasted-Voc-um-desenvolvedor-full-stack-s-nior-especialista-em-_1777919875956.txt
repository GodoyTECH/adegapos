Você é um desenvolvedor full-stack sênior especialista em sistemas web de PDV, gestão de estoque, dashboards administrativos, sistemas SaaS, banco de dados PostgreSQL, aplicações responsivas e integração com câmera do celular para leitura de código de barras.

Quero que você crie um sistema web completo chamado inicialmente de “Adega Smart POS”, voltado para adegas modernas no Brasil, especialmente adegas de São Paulo que vendem bebidas, combos, copões preparados, gin, whisky, vodka, Chanceler, energéticos, gelo, refrigerantes, cervejas, destilados, tabacaria se necessário e produtos similares.

O objetivo é criar um MVP profissional, funcional, bonito, responsivo e pronto para apresentação a clientes reais. O sistema precisa ser web, não instalável. Ele deve funcionar bem no computador do caixa e também no celular. O grande diferencial do sistema será substituir o leitor de código de barras físico, a pistola laser, pela câmera do celular ou pela câmera do dispositivo usando leitura de código de barras diretamente no navegador.

STACK OBRIGATÓRIA

Use:

- Front-end: React, Vite ou Next.js, preferencialmente uma estrutura simples e rápida para deploy.
- Estilização: TailwindCSS.
- Banco de dados: PostgreSQL usando Neon.
- Deploy: Netlify.
- ORM: Prisma, Drizzle ou integração SQL bem organizada. Escolha a opção mais estável para o projeto.
- Autenticação: login com e-mail e senha, sessão segura e controle de permissões por perfil.
- Scanner de código de barras: usar a câmera do navegador.
- Leitura de código de barras: implementar primeiro com BarcodeDetector se disponível no navegador e fallback com biblioteca ZXing, QuaggaJS ou alternativa confiável para browser.
- Gráficos: Recharts, Chart.js ou biblioteca similar.
- Armazenamento de imagens: deixar preparado para Cloudinary futuramente. No MVP, imagem pode ser opcional e pode funcionar com URL externa ou campo vazio.

IMPORTANTE SOBRE O SISTEMA

O sistema será usado por adegas que precisam vender rápido no balcão, controlar estoque, cadastrar produtos com código de barras, vender produtos compostos e acompanhar o faturamento em tempo real.

Não criar aplicativo instalável nativo. Criar uma aplicação web responsiva. Se possível, deixar preparado como PWA futuramente, mas isso não é prioridade agora.

4. SCANNER DE CÓDIGO DE BARRAS COM CELULAR

Implementar leitura de código de barras usando câmera do navegador.

Fluxo:

- usuário clica em “Escanear com câmera”;
- abrir modal ou tela com câmera ativa;
- solicitar permissão da câmera;
- detectar código de barras;
- ao detectar, parar leitura ou confirmar leitura;
- buscar produto no banco pelo código;
- se encontrado, adicionar ao carrinho;
- se não encontrado, mostrar mensagem:
  “Produto não cadastrado. Deseja cadastrar agora?”

Para produto não cadastrado:

- se usuário for ADMIN ou GERENTE/ESTOQUISTA, permitir cadastro rápido;
- se usuário for ATENDENTE, mostrar mensagem:
  “Produto não cadastrado. Chame um administrador ou gerente para cadastrar.”

Cadastro rápido deve ter:

- código de barras já preenchido;
- categoria;
- preço de venda;
- preço de custo opcional;
- estoque inicial;
- estoque mínimo;
- unidade;
- botão salvar e adicionar ao carrinho.

Fallback obrigatório:

Se a câmera não funcionar ou o navegador não suportar leitura automática:

- permitir digitar o código de barras manualmente;
- permitir buscar pelo nome;
- mostrar mensagem clara:
  “Não foi possível acessar a câmera. Digite o código ou busque pelo nome.”

Ingredientes:
- Gin: 150 ml;

6. Finalizar com condimento/decorativo.

13. REGRAS DE BAIXA DE ESTOQUE

Produto simples:

Ao vender produto simples, baixar o estoque do próprio produto.

Produto composto:

Ao vender produto composto, buscar todos os itens da ficha técnica em composite_product_items e baixar o estoque de cada ingrediente conforme quantity_used multiplicado pela quantidade vendida.

Se vender 2 copões, baixar o dobro dos ingredientes.

Se estoque insuficiente:

- mostrar alerta;
- no MVP, permitir configurar se bloqueia venda ou permite vender com estoque negativo;
- por padrão, alertar e permitir apenas para administrador/gerente;
- atendente deve ser bloqueado ou receber aviso para chamar gerente.

19. VALIDAÇÕES IMPORTANTES

Produto:

- nome obrigatório;
- preço de venda obrigatório;
- categoria obrigatória;
- se tiver código de barras, ele deve ser único por loja;
- estoque não pode ser vazio;
- tipo do produto deve ser simple ou composite.

Produto composto:

- precisa ter ao menos um ingrediente;
- precisa ter modo de preparo ou observação;
- precisa calcular custo baseado nos ingredientes;
- precisa permitir editar receita depois.

Venda:

- não permitir finalizar venda vazia;
- total precisa ser maior que zero;
- forma de pagamento obrigatória;
- caixa precisa estar aberto, se controle de caixa estiver ativo.

Estoque:
